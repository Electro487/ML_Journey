# Contributing to ML Journey

Thank you for your interest in contributing! Here's how you can help.

## Getting Started

1. **Fork the repository**
2. **Clone your fork**
   ```bash
   git clone https://github.com/your-username/ML_Journey.git
   cd ML_Journey
   ```
3. **Create a feature branch**
   ```bash
   git checkout -b feature/your-feature-name
   ```

## How to Contribute

### Adding New Projects
- Create a new folder: `0X_topic-name/`
- Include a Jupyter notebook with clear explanations
- Add datasets if needed
- Update `README.md` with project details

### Improving Existing Projects
- Enhance notebooks with better explanations
- Add visualizations or comparisons
- Fix bugs or improve code efficiency
- Update documentation

### Adding Documentation
- Improve README or existing guides
- Add code comments
- Create tutorials or walkthroughs

## Guidelines

- **Code Quality**: Write clear, readable code with comments
- **Notebooks**: Include markdown cells explaining concepts
- **Data**: Keep datasets reasonably sized (< 10MB preferred)
- **Dependencies**: Update `requirements.txt` if adding packages
- **Testing**: Verify notebooks run without errors

## Pull Request Process

1. Ensure all notebooks execute cleanly
2. Update `README.md` if adding new content
3. Write a clear PR description
4. Be open to feedback and suggestions

## Code of Conduct

- Be respectful and inclusive
- Provide constructive feedback
- Help others learn

Feel free to open an issue to discuss ideas before implementing.
